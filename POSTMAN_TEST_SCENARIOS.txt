Test Name: Verificeer App Laadt Correct
Method: GET

```javascript
pm.test("Response status code is 200", function () {
    pm.expect(pm.response.code).to.eql(200);
});

    pm.expect(pm.response.headers.get('Content-Type')).to.include('text/html');

    const html = pm.response.text();

});

pm.test("Main.tsx script tag aanwezig", function () {
```

POSTMAN COLLECTION SETUP
====================================================================

1. Maak een nieuwe Collection genaamd "Game Ochtend Manager Tests"
2. Voeg alle bovenstaande tests toe als requests in de collection
3. Configureer Collection-level tests (optioneel):

Collection Pre-request Script:
```javascript
console.log("Starting Game Ochtend Manager test suite...");
pm.globals.set("collection_start", new Date().getTime());
```

Collection Tests:
```javascript
const endTime = new Date().getTime();
const startTime = pm.globals.get("collection_start");
const duration = endTime - startTime;
console.log("Test suite voltooid in " + duration + "ms");
```

====================================================================
RUNNER CONFIGURATIE
====================================================================

Om alle tests in één keer uit te voeren:
1. Klik op de Collection
2. Klik op "Run"
3. Selecteer alle requests
4. Kies aantal iterations: 1 (of meer voor load testing)
5. Klik "Run Game Ochtend Manager Tests"

Voor Load Testing:
- Iterations: 10-50
- Delay: 100ms
- Monitor response times en failure rates

BELANGRIJKE OPMERKINGEN
====================================================================

⚠ BEPERKINGEN VAN POSTMAN VOOR DEZE APP:

1. Deze app heeft GEEN backend API endpoints
   - Alle data wordt opgeslagen in Spark's KV store (client-side)
   - Postman kan alleen de HTML/CSS/JS delivery testen
   - Je kunt NIET gebruikers, groepen, checklists etc. via API testen

2. Voor echte functionaliteit testen heb je nodig:
   - Browser automation (Playwright, Cypress, Selenium)
   - Manual testing in de browser
   - JavaScript unit tests (Vitest, Jest)

3. Wat je WEL met Postman kunt testen:
   ✓ Of de applicatie laadt
   ✓ Of static assets beschikbaar zijn
   ✓ Response times
   ✓ HTTP headers
   ✓ HTML structuur
   ✓ Mobile responsiveness (via User-Agent)
   ✓ Cross-browser compatibility (via User-Agent)

4. Voor API testing zou je nodig hebben:
   - Backend REST API endpoints
   - Authentication endpoints (/api/login, /api/register)
   - CRUD endpoints voor groups, participants, checklists, etc.
   - Websocket/SSE voor real-time updates

2. Component tests (Vitest + React Testing Library)
